### 2. Auth Service Entities (250 lines)

**Table: users**
```
Purpose: Store user account information
Columns:
- id (UUID, PK)
- email (VARCHAR(255), UNIQUE, NOT NULL)
- password_hash (BYTEA, NOT NULL) - encrypted
- first_name (VARCHAR(100))
- last_name (VARCHAR(100))
- date_of_birth (DATE, NULL)
- phone_number (VARCHAR(20), NULL)
- created_at (TIMESTAMP, NOT NULL, DEFAULT NOW())
- updated_at (TIMESTAMP, NOT NULL, DEFAULT NOW())
- is_active (BOOLEAN, DEFAULT true)

Indexes: email (UNIQUE), created_at
Relationships: One-to-many with user_roles, workouts, diet_logs, weight_logs
```

**Table: roles**
```
Purpose: Define user roles
Columns:
- id (INT, PK)
- name (VARCHAR(50), UNIQUE) - admin, trainer, user
- description (TEXT)
- created_at (TIMESTAMP)

Values: admin (full platform access), trainer (can manage assigned users), user (basic access)
```

**Table: permissions**
```
Purpose: Define granular permissions
Columns:
- id (INT, PK)
- resource (VARCHAR(100)) - users, exercises, workouts, etc.
- action (VARCHAR(50)) - read, write, delete
- description (TEXT)

Relationships: Many-to-many with roles via role_permissions
```

**Table: user_roles** (Junction Table)
```
Purpose: Map users to roles
Columns:
- user_id (UUID, FK to users)
- role_id (INT, FK to roles)
- assigned_at (TIMESTAMP)

Composite PK: (user_id, role_id)
```

**Table: sessions**
```
Purpose: Track active sessions
Columns:
- id (UUID, PK)
- user_id (UUID, FK to users)
- jwt_token (TEXT)
- ip_address (VARCHAR(45))
- user_agent (TEXT)
- created_at (TIMESTAMP)
- expires_at (TIMESTAMP)
- last_activity (TIMESTAMP)

Indexes: user_id, expires_at
Relationships: Many-to-one with users
```

### 6. Query Examples (150 lines)

**Common Queries:**

1. Get user profile with roles
```sql
SELECT u.*, array_agg(r.name) as roles
FROM users u
LEFT JOIN user_roles ur ON u.id = ur.user_id
LEFT JOIN roles r ON ur.role_id = r.id
WHERE u.id = $1
GROUP BY u.id;
```

2. Get user's recent workouts
```sql
SELECT w.*, count(we.id) as exercise_count, 
       sum(we.weight * we.sets * we.reps) as total_volume
FROM workouts w
LEFT JOIN workout_exercises we ON w.id = we.workout_id
WHERE w.user_id = $1 AND w.completed = true
ORDER BY w.start_time DESC
LIMIT 10;
```

3. Get user's weight trend over last 30 days
```sql
SELECT log_date, weight_kg,
       AVG(weight_kg) OVER (ORDER BY log_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) as avg_7day
FROM weight_logs
WHERE user_id = $1 AND log_date >= CURRENT_DATE - INTERVAL '30 days'
ORDER BY log_date;
```

### 7. Data Integrity Rules (100 lines)

```
- user.email must be unique
- user.password_hash cannot be null
- workout.start_time must be before workout.end_time
- diet_logs calories must be > 0
- weight_logs weight_kg must be > 0
- goals.target_date must be after goals.start_date
- All timestamps use UTC
```

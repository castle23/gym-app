## Contributing

Contributing to the project:
1. Follow coding standards (see [Development Guide](../development/))
2. Include tests with all changes
3. Update documentation as needed
4. Follow commit message conventions
5. Submit pull request with description

See root-level CONTRIBUTING.md for full guidelines.

## Reporting Issues

Found an issue? Report it with:
1. Description of the problem
2. Steps to reproduce
3. Expected vs. actual behavior
4. Environment details
5. Error messages/logs

Use the bug report template in **04-templates.md**.

## Feature Requests

Suggesting a feature?
1. Describe the feature
2. Explain the use case
3. Suggest implementation approach
4. Consider impact on other services

Use the feature request template in **04-templates.md**.

## Documentation Style

When writing documentation:
- Use clear, concise language
- Include code examples
- Use markdown formatting
- Link to related documents
- Keep information current
- Add TOC for long documents

See [Development Guide](../development/03-coding-standards.md) for style guidelines.
